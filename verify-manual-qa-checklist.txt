## Step 6: Frontend Loads Without Errors

**Goal**: Verify landing page renders cleanly without console/network errors

### Instructions:
1. Open Chrome DevTools: Press F12
2. Go to Console tab - should be clean (no red errors)
3. Go to Network tab - filter by XHR/Fetch requests
4. Navigate to http://localhost:3000 in a fresh tab
5. Wait for page to fully load (check Network tab - all requests should be green 200s)
6. Check Lighthouse score: Run Lighthouse audit (DevTools > Lighthouse > Analyze page load)

### Acceptance Criteria:
- [ ] **No console errors** (only warnings acceptable)
- [ ] **All network requests successful** (status 200/304)
- [ ] **3D globe renders** without WebGL errors
- [ ] **Page load time < 3 seconds** (from Network tab)
- [ ] **Hero section visible** with clear typography
- [ ] **Navigation bar sticky** and responsive

## Step 8: Results Feel Like Investigation

**Goal**: Verify that analysis results present findings as an investigation

### Instructions:
1. From analyzer page, paste this test job posting:
   ```
   Title: Remote Data Entry - $500/week
   Company: GlobalWorks Inc.
   We are looking for remote workers! No experience needed. Work from home and earn $500-$1000 per week. 
   Click the link below to apply and get started immediately. Limited spots available!
   Requirements: Must be 18+. No specific skills needed.
   To apply: Send $99 registration fee to activate your account.
   ```
2. Click Submit/Analyze
3. Wait for results to load
4. Inspect the results page

### Acceptance Criteria:
- [ ] **Risk/Scam Score displayed prominently** (visual indicator: color, number, gauge, or bar)
- [ ] **Score is accurate** (test posting should have HIGH risk - red/warning color)
- [ ] **Key red flags highlighted** (e.g., "No experience needed", "Work from home", "$$ promise", "registration fee required")
- [ ] **Each flag has confidence/reason** next to it
- [ ] **Scam network graph visible** (if applicable - connections to similar postings)
- [ ] **Detailed report section** scrollable with structured findings
- [ ] **Color coding used** (red = high risk, orange = medium, green = low)
- [ ] **Visual hierarchy clear** - most important info at top

### UX Feel Test:
- Does the results page feel like an **investigative report** rather than generic analytics?
- Would a user feel **confident** making a decision based on this information?


#### Result:
- [ ] PASS - Feels like investigation, clear findings

## Step 10: Charts Display and Render Correctly

**Goal**: Verify all data visualizations render and behave correctly

### Instructions:
1. Keep DevTools open on Console tab
2. Scroll through dashboard and analyzer results pages
3. Watch for any WebGL/Canvas/Chart errors in Console
4. Test interactions with each chart

### Acceptance Criteria:

**Rendering**:
- [ ] **No WebGL errors** in console
- [ ] **No Canvas errors** in console
- [ ] **All charts render** without errors
- [ ] **Fonts and labels visible** on all charts

**Pie Chart** (Risk Distribution):
- [ ] Renders without errors
- [ ] Slices are color-coded
- [ ] Legend appears below/beside
- [ ] Hover tooltip shows label + percentage
- [ ] Responsive - maintains proportions on resize

**Trend Chart** (Time Series Line):
- [ ] Renders with axes labeled
- [ ] Grid lines visible (if applicable)
- [ ] Line connects data points smoothly
- [ ] Hover shows tooltip with date and value
- [ ] Responsive - scales on resize

**Indicators Table**:
- [ ] Text readable
- [ ] Sortable (if applicable - check column headers)
- [ ] Data not truncated

**Performance**:
- [ ] Pie chart renders in **< 1s** after data loads
- [ ] Trend chart renders in **< 1s** after data loads
- [ ] No lag when hovering over charts
- [ ] Page doesn't freeze when switching between charts
